# BirdScope 前端页面评价报告（项目经理视角）

评估日期：2026-06-16  
评估对象：BirdScope React + Cesium + ECharts 前端页面  
评估方式：本地启动前端、后端与数据库后进行手工联调和浏览器交互验证

## 一、评估结论

当前前端已经具备项目演示和阶段验收的主体形态：页面信息架构清晰，核心功能围绕“空间范围选择、物种筛选、观测记录查询、地图展示、统计图表”展开，能体现项目的 GIS 与生态观测数据可视化特色。

在前后端联通后，核心链路可以跑通：选择“中国范围”样例后执行查询，页面返回 2000 条观测记录；选择 `Columba livia` 后再次查询，返回 1728 条记录；物种排行、月度趋势、区域统计相关接口均可从后端取得数据。整体已达到“可演示、可说明项目价值”的阶段。

但当前页面距离稳定交付仍有明显差距，主要风险集中在产品语义、异常提示、数据质量、移动端性能和统计视图逻辑一致性上。建议将当前状态定义为“联调通过但待产品打磨”，不宜直接作为最终版提交。

## 三、主要优点

1. 页面定位明确  
   三栏布局将查询条件、地图工作区、统计面板放在同一工作台中，符合 GIS 数据探索工具的使用心智。

2. 技术展示度高  
   Cesium 三维地球、WMS 热力图、局部观测点、ECharts 图表同时出现，能直观展示项目技术栈和数据可视化能力。

3. 核心数据链路已打通  
   后端启动后，观测记录查询、物种搜索和统计接口均能响应，说明前后端接口基础联调是有效的。

4. 查询结果反馈及时  
   点击执行查询后，左侧自动切换到“查询结果”，并显示结果总数和记录卡片，减少了用户查找结果的成本。

5. 基础响应式布局可用  
   在 390px 移动端宽度下未发现横向溢出，页面可按纵向顺序展示。

## 四、主要问题与风险

### P0：样例范围命名与实际查询结果不一致

“中国范围”按钮对应 bbox 为 `70,20,140,55`，实际覆盖中国及周边大范围区域。查询结果中出现俄罗斯、印度、日本、尼泊尔等记录。  

这会直接影响项目可信度。用户看到“中国范围”却得到大量非中国记录，会误以为筛选逻辑错误。

建议：

- 若目标是中国行政范围，应改用中国边界 polygon 或后端增加 `country_code=CN` 条件。
- 若只是东亚/亚洲样例，应将按钮名称改为“东亚样例范围”或“亚洲迁徙带样例”。

### P1：异常提示仍偏技术化

后端未启动时，统计区和物种搜索区显示 `Failed to fetch`。虽然不是静默失败，但对非技术用户没有解释价值。

建议：

- 改为“无法连接后端服务，请确认 FastAPI 已启动”。
- 在图表、物种搜索、查询结果三个区域分别提供上下文提示。
- 开发环境可附带接口地址，正式演示环境不直接暴露技术错误。

### P1：选中单一物种后，“物种排行”视图产品逻辑不清

选中 `Columba livia` 后，右侧仍默认停留在“物种排行”。从业务语义看，单一物种下继续展示“前 8 种观测最多物种”会让用户困惑。

建议：

- 选中物种后默认切换到“月度趋势”。
- 或将“物种排行”替换为“同范围高频物种对比”，并明确是否受当前物种筛选影响。

### P1：数据编码质量影响专业观感

区域统计接口返回 `SÃ£o Paulo`、`CÃ³rdoba` 等编码异常文本。前端如果直接展示，会削弱数据可信度。

- 在数据入库或 API 层修复字符编码。
- 前端报告中可暂时规避乱码字段，但根因应在数据管道处理。

### P2：地图重资源场景存在性能风险

桌面端 Cesium 和热力图渲染正常，但移动端截图捕获超时。虽然页面 DOM 没有横向溢出，但地图在小屏或性能较弱设备上的稳定性仍需专项验证。

建议：

- 移动端降低默认地图渲染压力。
- 对 Cesium 初始化、WMS 加载和点位渲染设置加载状态。
- 限制移动端一次性渲染的点数量。

### P2：查询条件与结果状态保留策略需要明确

执行查询后再回到“查询条件”，结果数量徽标会保留，这是有用的；但物种搜索选择后候选项区域有残留感，容易让用户误解是否已完成选择。

建议：

- 选择物种后收起候选列表，仅显示“当前物种”标签。
- 给结果数量徽标增加更新时间或当前筛选摘要。

## 五、项目管理建议

建议将后续工作分为三个阶段：

1. 验收前修复
   - 修正“中国范围”命名或查询逻辑。
   - 优化 `Failed to fetch` 错误提示。
   - 修复区域名称编码异常。
   - 明确单物种状态下的统计视图。

2. 演示增强
   - 准备 2 到 3 个稳定样例：全国/东亚范围、北京局部、多物种对比。
   - 增加“当前筛选摘要”，让评委快速理解结果来自哪些条件。
   - 给 GeoJSON 按钮增加真实导出或禁用态说明。

3. 交付质量提升
   - 增加前端冒烟测试脚本。
   - 增加 API 健康状态提示。
   - 针对移动端地图性能做专项测试。
   - 建立数据清洗校验清单。

## 六、验收判断

从项目经理视角看，当前页面可以用于阶段性展示，但还不适合作为最终交付版本。若用于课程答辩或项目演示，建议先完成 P0 和 P1 问题修复，尤其是范围命名、错误提示、统计语义和数据编码。
