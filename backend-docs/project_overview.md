# BirdScope 项目概述

> 来源整合：BirdScope_开题报告.md  
> 最后更新：2026-06-06

---

## 项目背景

基于 eBird/GBIF 全球鸟类观测数据，构建面向观鸟爱好者与生态研究者的在线三维观鸟地图平台，实现鸟类观测数据的空间展示、多维查询与时空分析。

数据源：GBIF Simple CSV 下载（`0009321-260519110011954.csv`），约 15GB，UTF-8 制表符分隔 TSV，约 2765 万条观测记录，数据年份 2024 年 8–11 月迁徙季。

---

## 基本目标

1. **鸟类观测数据地图展示** — 分级渲染：全球大尺度热力图 / 中等尺度区域聚合 / 局部矢量点
2. **多维组合查询** — 物种、时间、空间（地图框选、缓冲区、行政区划）
3. **时空动态可视化** — 时间滑块和动画播放，展示观测分布随月份变化
4. **数据统计图表** — 物种排行、月度趋势、区域统计（ECharts）
5. **GeoServer 图层管理** — 前后端交互实现图层发布、样式切换
6. **制图导出** — 地图截图 + 图面排版

## 拓展目标

- 候鸟迁徙路径可视化（按月质心连线，非 GPS 轨迹）
- 卷帘 / 分屏对比
- 物种信息卡片
- 大语言模型接入

## 功能模块

| 模块 | 核心功能 | 说明 |
|------|----------|------|
| 地图主场景 | Cesium 三维地球分级渲染 | 全球热力图 (WMS) / 局部矢量点 (FastAPI GeoJSON) 缩放切换 |
| 多维查询 | 物种检索、时间筛选、空间筛选 | 物种名搜索、日期选择器、地图绘制多边形、点选缓冲区 |
| 时空动态 | 时间滑块动画 + 卷帘对比 | 预加载分月聚合数据，滑动时本地切换或重新请求 |
| 统计图表 | ECharts 展示 | 联动查询结果，更新图表 |
| 图层管理 | GeoServer REST API 管理 | 图层增删、样式切换、发布控制 |
| 制图导出 | 地图截图 + 图面排版 | html2canvas 截图，Canvas 叠加图例 / 比例尺 |

## 技术架构概览

```
eBird/GBIF 批量数据 (15GB TSV)
    │
    ▼
Python 数据清洗、降采样（scripts/prepare_global.py）
    │
    ▼
PostGIS（PostgreSQL 16 + PostGIS 3.4，数据库 birdscope）
    │
    ├──▶ GeoServer 2.28.1 ──▶ WMS 图层 ──┐
    │                                      │
    └──▶ FastAPI（业务查询 / 空间分析 /    │
         GeoServer REST 管控）             │
              │                            │
              ▼                            ▼
         React + CesiumJS + Tailwind CSS + ECharts
```

**关键技术**：Python 3.11+、FastAPI、SQLAlchemy 2.x、PostGIS、GeoServer 2.28.1、React、CesiumJS、ECharts、GitHub 协作。

## 重要数据约定

- 迁徙路径展示的是"各月观测重心的连线"，**不是**单只鸟的 GPS 轨迹，原始数据不支持个体追踪。
- `species` 字段在约 1.4% 记录中为 NULL（仅匹配到属级），展示时使用 `scientific_name` 作为 fallback。
- `individual_count` 在约 5–6% 记录中为 NULL，热力图按记录数计，数量图须注明缺失。
