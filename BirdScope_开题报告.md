# 全球观鸟地图平台 —— 开题汇报

## 一、项目背景

基于eBird全球鸟类观测数据，构建一个面向观鸟爱好者与生态研究者的在线三维观鸟地图平台，实现鸟类观测数据的空间展示、多维查询与时空分析。

---

## 二、项目目标

### 基本目标

1. 鸟类观测数据地图展示
   分级渲染：
   全球大尺度：全球热力图
   中等尺度：区域聚合
   局部小尺度：局部矢量点
2. 多维组合查询
   物种、时间、空间（地图框选、缓冲区、行政区划）
3. 时空动态可视化
   时间滑块和动画播放，展示观测分布随月份变化
4. 数据统计图表
   物种排行、月度趋势、区域统计等
5. GeoServer图层管理
   前后端交互实现图层发布、样式切换
6. 制图导出

### 拓展目标

- 候鸟迁徙路径可视化
- 卷帘/分屏对比
- 物种信息卡片
- 大语言模型接入

## 三、功能模块

| 模块       | 核心功能                     | 说明                                                   |
| ---------- | ---------------------------- | ------------------------------------------------------ |
| 地图主场景 | Cesium三维地球分级渲染       | 全球热力图(WMS) / 局部矢量点(WFS+API) 缩放切换         |
| 多维查询   | 物种检索、时间筛选、空间筛选 | 物种名搜索、日期选择器、地图绘制多边形、点选设置缓冲区 |
| 时空动态   | 时间滑块动画 + 卷帘对比      | 预加载分月聚合数据至前端缓存，滑动时本地切换           |
| 统计图表   | ECharts展示                  | 联动查询结果，更新图表                                 |
| 图层管理   | GeoServer REST API管理       | 图层增删、样式切换、发布控制                           |
| 制图导出   | 地图截图 + 图面排版          | html2canvas截图，Canvas叠加图例/比例尺                 |

## 四、技术架构

```
eBird批量数据 (TSV)
    │
    ▼
Python 数据清洗、降采样
    │
    ▼
PostGIS (空间数据库)
    │
    ├──▶ GeoServer ──▶ WMS/WFS服务 ──┐
    │                                  │
    └──▶ FastAPI (业务查询/空间分析     │
         /GeoServer REST管理)          │
              │                        │
              ▼                        ▼
         React + Cesium + Tailwind CSS + ECharts
```

**关键技术点：**

- 前端：React + CesiumJS + Tailwind CSS + ECharts
- 后端：FastAPI
- 数据库：PostgreSQL + PostGIS
- 地图服务：GeoServer (WMS/WFS)
- 数据源：eBird Basic Dataset  2024年8-11月迁徙季
- 代码管理：GitHub
